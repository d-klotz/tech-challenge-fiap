    Explicação Prática:
        •	O MAE mostra o erro médio absoluto que o modelo comete nas previsões. É uma métrica simples e direta.
        •	O MSE penaliza grandes erros mais fortemente do que o MAE, sendo útil para identificar a presença de outliers que podem estar influenciando o desempenho do modelo.
        •	O RMSE é uma forma corrigida do MSE, facilitando a interpretação ao trazer os erros de volta à escala original dos dados.
        •	O R² Score indica a qualidade geral do ajuste do modelo aos dados: quanto maior, melhor o modelo está capturando a relação entre as variáveis.

    3 - Explicar essa etapa (kfold vs simplemente 5)

    Utilizar o KFold permite maior controle sobre a divisão dos dados, enquanto o uso direto de cv=5 simplifica a chamada. Ambas as abordagens fornecem resultados semelhantes, mas o uso de KFold oferece mais flexibilidade, como o controle sobre o embaralhamento dos dados.

    # Os valores de MSE indicam a soma dos quadrados das diferenças entre
     o valor real e o valor previsto pelo modelo. Quanto menor o MSE,
      melhor o modelo consegue prever o valor real. O MSE não tem relação
       direta com os valores previstos em si. O MSE não deve ser próximo do valor previsto, mas sim próximo de zero para indicar que o modelo está cometendo poucos erros. Ele serve como uma métrica para avaliar a precisão das previsões, e não como um valor para ser comparado diretamente às previsões. Quanto mais próximo o MSE estiver de zero, melhor é o desempenho do modelo.

    # Um MSE alto indica que o modelo está errando significativamente, ou seja,
     as previsões estão longe dos valores reais.

    # Um MSE muito alto indica a presença de outliers (valores que estão muito
     longe dos valores normais).

    # Se o MSE eleva ao quadrado as diferencas entre os valores
     previstos e os valores reais, o RMSE faz a raiz do MSE, trazendo
      o erro medio de volta para o valor original. Quanto menor o RMSE,
       melhor o modelo consegue prever o valor real.

    # O R2 Score é uma medida de quão bem as previsões do modelo se ajustam
     aos dados. Quanto mais próximo de 1, melhor o modelo. Quanto mais próximo
      de 0, pior o modelo. O R2 Score não tem relação direta com os valores
       previstos em si, mas sim com os valores reais. Se o R2 Score estiver
        próximo de 1, o modelo está prevendo o valor real mais precisamente

    1 -  verificar a necessidade de usar o .score vs r2_

    •	Aplicabilidade: Para outros modelos (como classificadores), o .score() geralmente retorna outras métricas, como a acurácia. Já r2_score só faz sentido para modelos de regressão.
    •	Para regressão, ambos funcionam de forma similar, mas se você já tiver as previsões (y_pred), pode ser mais natural usar r2_score.
        •	Se você quiser um atalho e não tiver as previsões, pode usar .score().
